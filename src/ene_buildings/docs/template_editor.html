Editing Templates

Templates are the base to draw a building from. Installed templates are stored as .bldg files in the building template directory of the plugin directory.

To share a template with someone, just give them the .bldg file for that template. The template directory can be opened from Extensions > Eneroth Townhouse System > Template Editing > Open Template Directory.

While being edited a template is represented by a component. There are 3 ways to get such a component.

  • An existing template saved to the library can be opened from the template editing menu or toolbar .
  • The component can also be placed from Sketchup's native component browser if it has already been loaded for the model.
  • A new component can be created from the selected entities by clicking New in the template editing menu or toolbar .

To save the template to the library (and pack it as a .bldg file) click Save Template in the toolbar .

1 Main Volume

The main volume of the building must be drawn directly in the component to be recognized by the plugin. The two gable faces must point in negative and positive X. All other faces must be perpendicular to these two. The volume can have any length but the length its drawn to will be used when creating the preview image.

The side of the volume towards negative Y is considered the front side. The facade plane should intersect default ground level along the X axis

2 Template Info

The template info dialog is used to set the template attributes, such as ID and name.

2.1 General

Name - What the template should be called. So far I haven't come up with a naming convention so write anything you find relevant.

Modeler - The person saving the template. If the template is based on another person's template use your own name anyway but credit them in the description.

ID - Starts with a prefix based on your name to avoid name collisions and people accidentally overwriting each other files when sharing them. If you for instance use the prefix "ene" (reserved by me, Eneroth3) and then update the plugin the template might be overwritten by one I made that is included in the plugin and happens to have the same name. The preview image shows either a template saved by the currently written ID or a placeholder so if you accidentally chose an ID that already exists for your installed templates the preview for it is shown.

2.2 Alignment

Alignment tells if the sides of the building are typically aligned towards a street, a courtyard or another building (firewall).

Depth is measured between the facade planes and details such as bay windows and eaves can be ignored.

3 Part Info

The part info dialog is used to set attributes to the currently selected group or component, such as its positioning within the building.

3.1 Positioning

These settings changes the X coordinate where the group/component is placed (where its origin is). Y and Z coordinates will be kept as they are in model.

This option must be set for the part to be drawn at all.

Group/component axes can be displayed by going to Windows > Model Info > Components and tick the checkbox "Show component axes" to easier locate the origins.

3.1.1 Array

Margin and interval on resulting building.

Array is used to evenly spread a number of groups/components along the width of the building volume, either by a fixed number or by an approximate distance between.

Margins can be used to add a fixed distance at each side before spreading evenly over the remaining space.

The number of groups/components can be forced to either an even or odd value by setting rounding.

Dormer window not drawn when outside building.
Dormer window drawn anyway.

By default groups/components are not drawn if they are positioned partly besides the main building volume. This behavior can be overridden if undesired.

3.1.2 Align and centering

Places origin at the edge or in the middle of the volume. Left is towards negative X and right towards positive X.

3.1.3 Relative

Relation between distance from negative X (left) side and building width. E.g 0,25 is a quarter of the building width, starting from the left side.

3.2 Material Replacement

By default all materials on the main volume can be replaced from the building Properties dialog. By ticking "Replace materials within this part" the plugin will also check for and replace materials within this group. This is useful when a group has multiple materials to replace so painting the group/component itself doesn't work.

This option is only available for groups. Automatically modifying individual components would clutter the component browser with multiple versions of it. However the plugin allows you to easily convert a component to a group.

3.3 Solid Operations

Solid union and subtract can be performed with a group/component onto the main volume of the building.

Solid operations are slow to perform. If possible use cut-opening components instead.

Solid index is used to determine what order to perform the operations in starting at the lowest number. Default is 0.

3.3.1 Cut Multiple Faces

There's also an option to use the part's naked edges to cut openings, similar to the native cut opening feauture but on multiple faces at once. This is strictly isn't a solid operation and is always performed after the the real solid operations (union and subtract). This feature is useful when the part spans multiple faces, e.g. a bay window the roof eaves and continuing on the roof.
